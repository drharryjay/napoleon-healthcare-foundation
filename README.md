# Napoleon Healthcare Foundation Website

Responsive frontend for Napoleon Healthcare Foundation (NHF), a Nigerian community-health NGO (CAC/IT/NO 85319) focused on free medical outreach, preventive health education, screening, eye care, dental care, maternal and child health support, and community-based healthcare services.

**Live:** https://napoleonhealthcarefoundation.org (GitHub `main` → Netlify auto-deploy; DNS on Cloudflare)

## Stack

- Vite + React + TypeScript
- `react-router-dom` (every route code-split except Home)
- CSS custom properties, responsive component styles, motion system with full `prefers-reduced-motion` support
- Lucide React icons
- Netlify Forms (contact, partner, volunteer, newsletter)

## Run Locally

```bash
npm install
npm run dev
```

PowerShell on some Windows systems blocks `npm.ps1`. If that happens, run:

```bash
npm.cmd install
npm.cmd run dev
```

## Build

```bash
npm run build
```

The production output will be created in `dist/`.

## Editable Content

Most website content is stored in `src/data/` — see the fuller map in
`NHF Website Vault/02 Editable Content Map.md`:

- `siteConfig.ts`: organization details, contact info, social links, navigation, hero images
- `servicesData.ts`: service titles and descriptions (per-service photos map in `src/pages/WhatWeDoPage.tsx`)
- `impactData.ts`: impact metrics and full per-outreach records/stories/photos — **every headline figure's derivation is commented at the top of the file**; keep figures and their sources in sync
- `articlesData.ts`: health education categories and article cards
- `galleryData.ts`: gallery photos, captions, and lightbox data
- `partnersData.ts`: partner audiences, partnership options, donation categories
- `volunteerData.ts`: volunteer form dropdown options
- `trusteesData.ts`: trustee names/occupations and guiding principles

Publish only verified figures. Bank details and payment links must not be added until confirmed by NHF leadership.

## Photos

- `public/images/`: original outreach JPEGs; responsive WebP variants are generated by `node scripts/optimize-images.mjs` (add new filenames to the script's list first)
- `public/images/outreach/`: curated and per-event WebP photos, already web-sized, referenced directly
- Slideshows use `src/components/PhotoSlideshow.tsx` — pass a list of `{ src, alt }`; the parent element defines the box

## Forms

All four forms run on Netlify Forms with client-side validation, required consent, and email notifications. The static registration duplicates Netlify needs at build time live in `public/netlify-forms.html` — keep its field names in sync with `src/components/ContactForm.tsx`.

## Deployment

Pushing to `main` triggers a Netlify production deploy (`netlify.toml` sets build + security headers; `public/_redirects` returns real HTTP 404s for unknown URLs). Each production deploy consumes Netlify credits — batch commits and push once per body of work.
